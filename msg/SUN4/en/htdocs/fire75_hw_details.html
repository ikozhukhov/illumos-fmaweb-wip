The Message ID:  %MSGID% indicates a problem in maintaining a communication link between components in the PCI-Express subsystem. Use the fmdump -v -u EVENT_ID command to determine the faulty components as shown in the example below. 
fmdump -v -u 032a9c2c-e66c-61f2-d8e7-e5d8a7fb5b27
  TIME                 UUID                                 SUNW-MSG-ID
  Sep 29 05:04:20.7430 032a9c2c-e66c-61f2-d8e7-e5d8a7fb5b27 SUN4-8000-75
    50%  fault.io.fire.asic
          FRU: hc://product-id=SUNW,Sun-Fire-T200/component=IOBD
         rsrc: hc:///ioboard=0/hostbridge=0/pciexrc=0

    50%  fault.io.fire.pciex.device
          FRU: hc://product-id=SUNW,Sun-Fire-T200/component=IOBD
         rsrc: hc:///ioboard=0/hostbridge=0/pciexrc=0/pciexbus=2/pciexdev=0/pciexfn=0
Have this information available when contacting Sun Services for a repair.

NOTE - If this message occurs on a Sun Fire T2000, it may be because a required parameter has not been set in the /etc/system file. See the Sun Fire T2000 Server Product Notes available at the Sun Documentation Center for details (enter "Sun Fire T2000 Server Product Notes" in the Search Documentation box).
